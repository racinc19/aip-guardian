# AIP-Guardian Deep Overview

## Architecture

This directory contains deep-dive documentation for the AIP-Guardian project.

## Core Principles

1. **Executable First** — Every asset must be deployable and runnable
2. **Mission Aligned** — All components serve the AIP mission
3. **Transparent** — Open source, publicly verifiable
4. **Scalable** — Built to expand as the mission grows

## System Design

### Guardian Layer
The Guardian layer is the outermost operational interface. It handles:
- Public-facing deployment via GitHub Pages
- Entry point routing through index.html
- Documentation accessibility via markdown files

### Deep Layer
The deep layer (this directory) contains:
- Architectural specifications
- Extended documentation
- Future roadmap files
- Diagram assets
